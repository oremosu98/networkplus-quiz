TB v3 Phase 5 — founder dogfood

Trace mode smoke test

Walk these 10 steps end-to-end before signing off v6.2.0. Each step = one action + one observable outcome.

  1. Open the Topology Builder v3 page. Drop a star topology: 1 switch, 1 server, 3 workstations. Cable them all to the switch.
    Expected: 5 devices on the canvas, 4 cables, no warnings.
  2. Configure IPs: workstations 10.0.0.10/24, 10.0.0.20/24, 10.0.0.30/24 with gateway 10.0.0.1; server 10.0.0.100/24; switch needs no IP.
    Expected: All devices configured. No diagnostic drawer warnings.
  3. Click the Trace pill in the modebar (next to Simulate).
    Expected: Trace panel slides in from the right (240ms). Header shows TRACE eyebrow + "Trace mode" + × button. Controls section shows src/dst dropdowns + ping pill + Start trace / Next hop / Play / End trace buttons. Start disabled until src + dst are different.
  4. Pick source = WS-1, destination = SRV-1. Click Start trace.
    Expected: Packet appears at WS-1. Hop list populates with 3 chips (1·WS-1 current with accent left-bar, 2·SW-1 dim, 3·SRV-1 dim) cascading in 40ms apart. Annotation pane shows "Hop 1 of 3 / WS-1 / L7 · Application / Originates ICMP echo request / Targets 10.0.0.100 via default gateway 10.0.0.1". Next button enables.
  5. Click Next hop.
    Expected: Packet glides 250ms from WS-1 to switch (smooth cubic-bezier ease-in-out). 80ms accent glow on switch on arrival. Hop list: chip 1 muted ✓, chip 2 current with accent left-bar. Canvas badge "2" appears on switch. Annotation updates to "Hop 2 of 3 / SW-1 / L2 · Data Link / Forwards via MAC table / Egress port toward 10.0.0.100".
  6. Click Next hop again to reach the destination.
    Expected: Packet glides to SRV-1. Annotation: "Hop 3 of 3 / SRV-1 / L7 · Application / Receives ICMP echo, sends reply / Replies to 10.0.0.10". Next button disables (at destination). Mode = done.
  7. Click End trace to clear. Now pick source = WS-2, destination = WS-3. Click Start trace, then Play.
    Expected: Autoplay runs through the trace: 200ms pre-roll, then 600ms-per-hop glide with 120ms gap between hops. Play button text swaps to "Pause" while running. End-of-trace, mode = done.
  8. Click End trace. Now break WS-1: clear its config.gateway field (Inspector → Network → Gateway). Open Trace, pick WS-1 → SRV-1, click Start.
    Expected: Trace fails at hop 1. WS-1 shakes (200ms × 4 cycles) + glows red (1.2s fade). Annotation reason switches to red-tinted "Workstation-1 has no default gateway." Mode = paused (auto-paused on failure). Hop chip 1 shows red left-bar + red dot.
  9. Switch to Simulate mode (click Simulate pill — Trace closes). Re-trigger the failed ping (WS-1 → SRV-1). In the Simulate log, find the failed row.
    Expected: Failed log row shows "→ Trace" chevron at the end. Click the chevron. Simulate closes, Trace opens with WS-1/SRV-1 pre-populated, packet at source, hop list rendered. (One-click handoff from failure to investigation.)
  10. Close Trace via × button. Re-fix the WS-1 gateway. Open Trace again, verify the trace now succeeds end-to-end. Then press Esc to confirm Esc closes Trace.
    Expected: Fixed trace runs clean (no failure beat). Esc closes Trace cleanly (panel slides out 180ms — faster exit per asymmetric timing). No errors in browser console. All 5 rail panels (Inspector, Picker, Diagnostic, Simulate, Trace) close cleanly when another opens (cross-rail mutex symmetric).
If all 10 steps pass: v6.2.0 ready to ship. If any step trips, fix root cause + retest the affected steps before ship. Same discipline as Phase 4.
