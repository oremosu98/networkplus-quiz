# TB v3 Phase 7 — 3D mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Layer a perspective-shifted 3D view on Phase 5 Trace + Phase 6 OSI substrates. Topology renders as a table-top tilted scene; devices stand vertically; trace packet glides along cables then rises into standing devices where the Phase 6 OSI 7-row cascade fires inside the device face.

**Architecture:** Additive inside the existing `features/topology-builder-v3.js` IIFE — zero new modules, zero new dependencies, CSS 3D transforms only (no Three.js / WebGL / `vendor/three/*`). `state.mode = '3d'` already pre-reserved in the enum; the modebar 3D pill (locked since Phase 1) unlocks. `_open3D(payload?)` wraps `_openTrace` per Phase 6 OSI pattern. Phase 6's `_renderOSIPanel` 7-row generator extracts into a shared `_renderOSIStack` helper that both Phase 6 panel and Phase 7 in-device cascade call with different variant classes. Cross-rail mutex extends to 6 panels.

**Tech Stack:** Vanilla JS (IIFE in `features/topology-builder-v3.js`), scoped CSS (`features/topology-builder-v3.css`), `requestAnimationFrame` (emil §8.6 triple-timer cancel discipline), `@media (prefers-reduced-motion)` gates, Node UAT suite (`tests/uat.js`), Playwright E2E (`tests/e2e/app.spec.js`).

**Spec:** `docs/superpowers/specs/2026-05-24-tb-v3-phase7-3d-design.md`

**Branch:** `feat/tb-v3-phase7` (already created at commit `51a2215` — spec doc)

```javascript
  // ===========================================================================
  // Phase 7 Stage 1: _renderOSIStack
  // Shared 7-row HTML generator. Phase 6 calls with variant='panel' (renders
  // into the right-rail trace annotation panel). Phase 7 calls with
  // variant='in-device' (renders into the standing device card in 3D mode).
  // Both variants emit the SAME row DOM — only the wrapper class differs
  // for scoped CSS variant styling.
  // ===========================================================================
  function _renderOSIStack(layerStack, opts) {
    var variant = (opts && opts.variant) || 'panel';
    var wrapperClass = (variant === 'in-device')
      ? 'tb3-osi-stack tb3-osi-stack--in-device'
      : 'tb3-osi-stack';

    var rowsHtml = '';
    for (var i = 0; i < layerStack.length; i++) {
      var row = layerStack[i];
      var rowClasses = 'tb3-osi-layer';
      if (row.active) rowClasses += ' is-active';
      else rowClasses += ' is-passive';
      if (row.failure) rowClasses += ' is-failure';

      rowsHtml += '<div class="' + rowClasses + '" data-layer="' + row.layerNum + '"' +
                  ' aria-label="' + _escAttr('Layer ' + row.layerNum + ' ' + row.layerName + ' — ' + (row.active ? 'active' : 'passive')) + '">' +
                    '<span class="tb3-osi-layer-num">L' + row.layerNum + '</span>' +
                    '<span class="tb3-osi-layer-name">' + _escAttr(row.layerName) + '</span>' +
                    '<span class="tb3-osi-layer-proto">' + _escAttr(row.proto || '') + '</span>' +
                    '<span class="tb3-osi-layer-verb">' + _escAttr(row.verb || '') + '</span>' +
                  '</div>';
    }

    return '<ol class="' + wrapperClass + '">' + rowsHtml + '</ol>';
  }
```
