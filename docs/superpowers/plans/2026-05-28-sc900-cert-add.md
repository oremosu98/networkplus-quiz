## 1 — Spec (what + why)

### What
Add **Microsoft SC-900 (Security, Compliance & Identity Fundamentals)** as the **sixth cert family** in CertAnvil, alongside Network+ (N10-009), Security+ (SY0-701), AZ-900, AI-900, and A+ (Core 1 + Core 2). Pattern A subdomain: **`sc900.certanvil.com`** (a NEW subdomain — SC-900 is a third Microsoft role family, distinct from Azure infra and AI/data; see §2 #2 + §9 Q1). Pro-tier gate (anon visitors hit the landing diagnostic; switching INTO SC-900 in the cert app requires Pro). All cert-aware home/Progress/Analytics/Settings surfaces lift-and-shift unchanged (they read `CERT_PACK` + `CURRENT_CERT` + `TOPIC_DOMAINS` + `DOMAIN_WEIGHTS` at runtime).

### Why
- **Sixth cert / third Microsoft role family** — proves the Pattern A playbook across a third Microsoft track (Security/Compliance/Identity ≠ Azure infra ≠ AI/Data). The cert-pack abstraction is now validated single-cert (Net+/Sec+/AZ-900/AI-900), dual-exam (A+), and across 3 vendors × multiple role families.
- **Strong cross-sell from the existing portfolio** — VoC §11 confirms the dominant buyer journey is **AZ-900 → SC-900** within 1-2 months, and SC-900 pairs naturally with CompTIA Security+ (both already live). The audience overlap with current CertAnvil buyers is high.
- **High volume + clear differentiator** — SC-900 is a popular Microsoft fundamentals cert, and the VoC surfaces a sharp, defensible content gap: the "many Defenders" disambiguation problem + the Entra/Defender-XDR/Purview rebrand drift that lags in competitor prep material.
- **Personal goal** — founder continuing the Microsoft fundamentals track after AZ-900/AI-900.

### Non-goals
- **No new app architecture** — every cert-aware surface reads `CERT_PACK` at runtime. Adding SC-900 = populate one cert pack + wire Pattern A detection (now a 7-way ternary). Zero render-LOGIC change. Proven across v7.3.0 → v7.6.0.
- **No exam-format changes** — SC-900 is 700/1000 pass, ~40-60 Q / 60 min (see §2 #3). `CERT_PACK.meta` already handles per-cert exam shape.
- **No new monetization mechanics** — Pro-tier gate identical to AZ-900/AI-900 (visible in switcher, gated at switch time via `tadSwitchCert`).
- **No paid-bank content ingestion** — original exemplar authoring from public Microsoft Skills Measured + public Microsoft Learn modules only. **Skillcertpro is on the prohibited list** (VoC §6 flags active shilling + exam-dump red zone on SC-900 threads). Same legal discipline as all prior banks.

### Out-of-scope (deferred follow-ups)
- SC-900 Phase 3 retention-concept growth beyond the initial 8 — defer until founder hits gaps in real study (Jason Dion Method for Microsoft Learn / official MS practice tests).
- SC-900 GT tables (deterministic-fact tables) — Security/Compliance/Identity has candidate surfaces (the 6-Defender → workload mapping, the Entra license-tier → feature gating) but defer until a clear enumerable pattern emerges.
- **NO KQL content** — KQL is SC-200 (associate), explicitly NOT on SC-900 (VoC §10 rec #7, high confidence).
- Future SC-track certs (SC-200/SC-300/SC-400/SC-100) — see §9 Q1 for the subdomain-sharing decision; not built here.
