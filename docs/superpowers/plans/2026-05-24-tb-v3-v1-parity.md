## §4 Title-case sweep

ALL scenario titles (existing V3 ones + newly-ported V1 ones) get title-cased. Rule: capitalize the first letter of every word except certain short joiners (`a`, `and`, `the`, `of`, `with`, `on`, `in`, `or`, `for`).

Examples:
- `'Star topology with central switch'` → `'Star Topology With Central Switch'`
- `'3-tier hierarchical (core / distribution / access)'` → `'3-Tier Hierarchical (Core / Distribution / Access)'`
- `'Router-on-a-stick (inter-VLAN routing)'` → `'Router-on-a-Stick (Inter-VLAN Routing)'`
- `'DMZ / screened subnet'` → `'DMZ / Screened Subnet'`
- `'SOHO (router + switch + AP + endpoints)'` → `'SOHO (Router + Switch + AP + Endpoints)'`

(Acronyms stay uppercase. Hyphenated compound words get each component capitalized except the joiners.)

### Task 1.2: add one-shot migration for saved state

- [ ] **Step 1:** Find `_initTb` or wherever `state` is first hydrated from localStorage. Search:

Add migration function near other state helpers:

```javascript
  // ===========================================================================
  // V1 parity migration: rewrites old V3 device type ids to V1 conventions.
  // Runs once per hydration. Idempotent (safe to call multiple times).
  // ===========================================================================
  var _V1_TYPE_RENAMES = {
    'ap': 'wap',
    'workstation': 'pc',
    'vpn-gateway': 'vpg',
    'ids-ips': 'ids',
    'isp-modem': 'isp-router'
  };
  function _migrateStateTypesToV1() {
    if (!state || !Array.isArray(state.devices)) return;
    var renamed = 0;
    for (var i = 0; i < state.devices.length; i++) {
      var oldType = state.devices[i].type;
      if (_V1_TYPE_RENAMES.hasOwnProperty(oldType)) {
        state.devices[i].type = _V1_TYPE_RENAMES[oldType];
        renamed++;
      }
    }
    if (renamed > 0) {
      console.info('[tb-v3] V1 parity: migrated ' + renamed + ' device(s) to V1 type ids');
      _saveState();   // persist the migration so it runs once
    }
  }
```

## Stage 4: Title-case ALL scenario titles

**Subagent model:** Sonnet (invokes **stop-slop** for final review)

**Goal:** Apply title case to every scenario's `title` field. Rule: capitalize the first letter of every word except short joiners (`a`, `and`, `the`, `of`, `with`, `on`, `in`, `or`, `for`, `to`, `at`). Acronyms (DMZ, VPN, SVI, etc.) stay uppercase. Numbers + hyphenated compounds get each component capitalized.

### Task 4.1: write title-case helper + apply

- [ ] **Step 1:** Add a small helper near other string utilities:

```javascript
  // Title-case helper for scenario titles. Capitalizes first letter of each
  // word except short joiners. Preserves acronyms (UPPERCASE words stay so).
  var _TITLE_CASE_LOWER_WORDS = new Set(['a','an','and','the','of','with','on','in','or','for','to','at','vs','via']);
  function _titleCase(str) {
    if (!str || typeof str !== 'string') return str;
    return str.split(/(\s+|-)/).map(function (part, idx) {
      if (part.match(/^\s+$/) || part === '-') return part;
      // If already all-uppercase (>=2 chars) → preserve as acronym
      if (part.length >= 2 && part === part.toUpperCase() && /[A-Z]/.test(part)) return part;
      // Lower-word check (only for non-first words)
      if (idx > 0 && _TITLE_CASE_LOWER_WORDS.has(part.toLowerCase())) return part.toLowerCase();
      // Capitalize first letter
      return part.charAt(0).toUpperCase() + part.slice(1).toLowerCase();
    }).join('');
  }
```
