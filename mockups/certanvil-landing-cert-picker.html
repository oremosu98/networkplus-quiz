Design Decisions · Implementation Notes
Why a landing page (vs. straight-to-dashboard) With multi-cert ahead, customers need a way to discover what's available + opt into the cert they want. Going straight to Network+ would confuse anyone arriving for AZ-900 / CCNA / etc. A landing page also captures "Notify me" leads for upcoming certs (mailing list growth before each cert launches).
Builder-mode security model Three options for activation, ranked by my preference:
1. localStorage flag (nplus_builder_mode = 'true') — simple, persistent, no URL hint. Risk: anyone reading bundled JS sees the flag exists. Not a security boundary, just obscurity.
2. URL parameter (?builder=1) — bookmarkable, shareable across browsers. Risk: same as above, plus URL leakage in referrer headers.
3. Subdomain (builder.certanvil.com) — strongest separation, can be password-protected via Vercel. Most setup overhead.

For the "private to builder" model where Security+ is your study tool not a customer feature, option 1 + the existing secplus-quiz-sable.vercel.app separate-deploy is enough. Customers stay on Network+, you flip the flag in DevTools when you want the full cert picker.
Cert tile click behavior Two architectural options:
1. Subdomain hopping — click Network+ → navigate to networkplus.certanvil.com → URL host detection in app.js loads netplus pack. Simple, reuses existing detectCert logic. Recommended.
2. Single deploy with cert switcher — click Network+ → set localStorage flag → reload → app loads netplus pack. Adds complexity (mutable CURRENT_CERT) but slicker (no URL hop).

Option 1 is the path of least resistance — you already have two Vercel deploys, just need DNS subdomain setup. Landing page lives at root (its own minimal Vercel project, or a static page hosted at the existing Vercel project under a separate route).
Implementation scope (when ready to build) This is a SEPARATE deploy from the cert apps. New tiny Vercel project just hosting landing.html + a few assets. Routes:
• certanvil.com → landing project
• networkplus.certanvil.com → existing networkplus-quiz project
• securityplus.certanvil.com → existing secplus-quiz-sable project (private — no marketing link)
• www.certanvil.com → 301 redirect to root

Tonight's DNS migration just points certanvil.com at networkplus-quiz (current customer entry). When the landing page ships, repoint root → landing project.
Pricing positioning (placeholder for now) The hero copy avoids price commitment. The "Sign in" link in the topbar implies accounts but the current state doesn't require them. When you add paid tiers, the landing page will likely add: pricing card row (free / paid / lifetime), "Founding member" callout, social proof carousel.
What's NOT in this mockup yet Animations (subtle pulse on LIVE tile, skeleton loaders for "Coming soon"). Email capture form for "Notify me". Dark/light theme toggle in the topbar (currently dark only). Mobile hamburger nav (currently desktop layout). Account auth flow (sign in / sign up). Pricing/payment integration. All deferrable until the cert-mode SaaS launch trigger fires.
