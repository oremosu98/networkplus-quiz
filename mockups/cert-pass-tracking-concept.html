Implementation notes scope · schema · files · risks

Schema (Supabase)

Stored as a JSON map on the existing profiles.metadata jsonb column — no new table, no migration friction. Keys are cert IDs matching the cert pack (netplus, secplus, future).

// profiles.metadata.cert_results — keyed by cert ID { "netplus": { "status": "passed", // "passed" | "attempted" "score": 767, "max_score": 900, "pass_score": 720, // per-cert constant, helps "X above/below cutoff" copy "date": "2026-05-05", "centre": "Pearson VUE", // optional "created_at": "2026-05-08T14:32:00Z", "updated_at": "2026-05-08T14:32:00Z" }, "secplus": { ... } }

Per-cert score config (cert pack)

Each cert pack (certs/netplus.js, certs/secplus.js, etc.) declares its score format so the form + tile copy are cert-aware. Single score / max-score / pass-score triple per your default-G choice — works for CompTIA, AWS, Cisco %.

// certs/netplus.js (existing pack adds these 3 fields) window.CERT_PACKS.netplus.exam = { format: "scaled", // "scaled" | "percent" maxScore: 900, passScore: 720 }; // certs/secplus.js window.CERT_PACKS.secplus.exam = { format: "scaled", maxScore: 900, passScore: 750 }; // CCNA when shipped: { format: "percent", maxScore: 100, passScore: 82.5 }

Confetti reuse

  • Confetti modal reuses existing launchConfetti() from app.js (the v4.42.0 milestone celebration helper — 150 particles, 8 colors).
  • Score pill animates via existing animateCount(id, 0, score, 1200). Sub-second perceived latency.
  • Failed branch skips launchConfetti() entirely — different emotional tone.

Email (Pass branch only, async)

  • POST to landing/api/notify.js on Save success (existing Resend integration). Subject: "🎉 You passed Network+ · 767/900". Body: 2-paragraph encouragement + CTA back to platform.
  • Non-blocking. Modal fires immediately; email is fire-and-forget. 500-error from Resend just logs — doesn't break the user flow.
  • Failed branch: no email (avoids "you failed" inbox notifications that nobody wants).

Replaces hardcoded swap

  • Removes the v4.89.5 hardcoded "767/900" string in landing/auth.js:262. Cert-tile personalization becomes data-driven via profile.metadata.cert_results.netplus.
  • The "My certs" modal (v4.89.6) similarly becomes data-driven — currently hardcoded too.
  • Backwards-compat: anonymous visitors still see the pre-existing static cert-tile copy. Personalization only fires for signed-in users with a result row.

Pass Guarantee tie-in

  • The "Apply for Pass Guarantee →" CTA on the failed-branch modal links to a future Pass-Guarantee form. Until that ships, link to landing/pricing.html#pass-guarantee + a ?attempt=secplus&score=680 param so any future form can pre-fill.
  • Eligibility logic stays out of the cert-tracking ship — this is just data + UX. Pass Guarantee enforcement is a separate ticket.
