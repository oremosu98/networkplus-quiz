  • Storage: STORAGE.NA_MASTERY, NA_LESSONS, NA_STATS per the existing PORT_*/AB_*/OS_*/CB_* convention.

Q3 of 10 tcpdump filters Foundational
You need to capture only HTTPS traffic going TO the server at 10.0.0.50. Which tcpdump filter expresses this?
✓ Correct
tcpdump uses BPF (Berkeley Packet Filter) syntax — keywords like tcp port, dst host, joined by and/or/not. Option B is Wireshark's display filter syntax (different grammar — uses == and protocol.field dot-notation). Option C captures traffic FROM the server, not TO it. Option D's host matches both directions — too broad.

State 7 · Lessons mode · "tcpdump filter cheatsheet" Lesson 1 of 3
Lesson 1 of 3
tcpdump filter cheatsheet
BPF (Berkeley Packet Filter) syntax — what tcpdump uses on the command line
Step 2 of 5
Filter by host or network
tcpdump's host qualifier matches a specific IP — by default in BOTH directions (source OR destination). Add src or dst to narrow it.
tcpdump host 10.0.0.5 # traffic to OR from 10.0.0.5 tcpdump src host 10.0.0.5 # only FROM 10.0.0.5 tcpdump dst host 10.0.0.5 # only TO 10.0.0.5 tcpdump net 10.0.0.0/24 # whole subnet
Quick reference
host X Match traffic to OR from IP X
src host X / dst host X Direction-specific match
net X.X.X.X/Y Match a whole subnet
port N / src port N / dst port N Port match (any/source-only/destination-only)
tcp / udp / icmp / arp Protocol match
and / or / not Boolean combinators (also && / || / !)
