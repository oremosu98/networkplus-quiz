TB v3 Phase 7 v2 — 3D popup improvements mockup

Proposal for the next iteration. v6.4.1 ships the popup visible; this is what we'd do next to make it actually nice to look at.

What's in this mockup

  1. Core 1Per-device 3D illustrations matching the device type (router has antennae, switch has port LEDs, console has controller, server has rack units, etc.)
  2. Core 2Realistic material — gradients, soft shadows, beveled edges, ambient lighting
  3. Core 3Device labels (hostname + IP) visible below each device on the floor plane
  4. +Centroid offset — devices auto-center, fixes the bottom-right cluster bug
  5. +Floor grid — faint cream-on-cream, fades away from center (spatial reference)
  6. +Color-coded accent stripes per device family (network / endpoint / security / cloud)
  7. +Cable labels — port names + link type (Gig, 10G), only on hover
  8. +Cable pulse — very subtle traveling glint along cables (ambient liveness)
  9. +Improved initial camera — slightly lower tilt, tighter zoom
  10. +Fit-to-view button in header — auto-frames all devices on click
  11. +Device legend chip (collapsible) — shows family color coding
  12. +Hover state — device lifts slightly, tooltip shows IP/MAC/ports

1

Per-device illustrations

Each device type gets a recognizable 3D shape — router with antennae, switch with port LEDs, server with rack units, game console with controller. No more identical rectangles.

2

Realistic material

Top-face gradient + side-rim darkening + cast shadows + tiny LEDs that catch the light. Reads as a product illustration, not a CAD diagram.

3

Device labels (hostname + IP)

Each device has a floating label on the floor plane — hostname in Fraunces serif, IP in mono below. Labels counter-rotate so they stay readable as you spin the scene.

4

Floor grid + centered scene

Faint grid gives spatial reference; centroid-offset positions the topology around the scene origin so devices stay framed regardless of canvas coordinates.

5

Color-coded families

Tiny accent stripe on each device's base — bronze for network, warm grey for endpoints, red-bronze for security, blue-bronze for cloud. Subtle, brand-aligned.

6

Cable pulse + labels

Cables get a very subtle traveling glint (stroke-dasharray animation, 2.4s loop) suggesting active links. Hover a cable to see its port labels.

7

Fit-to-view + Reset

Two header buttons: Fit-to-view auto-frames all devices in the camera; Reset returns to default angle. Double-click still resets (existing behavior).

8

Device legend chip

Small floating chip in the bottom-right shows the family color coding. Collapsible. Helps users decode the visual language at a glance.

Per-device-type illustrations

10 device families. Each gets a distinct visual that reads as the device type from any rotation angle.

Implementation effort estimate: 5 device illustrations done in this mockup (Router · Switch · Workstation · Server · Game Console). Remaining 10+ device types follow the same recipe (~30-60min each). Plus: centroid offset (small JS change), cable labels on hover (small DOM + CSS), legend chip (small CSS), fit-to-view (camera math), color-family accent (CSS variable map). Roughly 1-2 days of focused work.

Tech: pure CSS 3D — no Three.js, no PNG assets, no new dependencies. Same engine as v6.4.0 — this is purely a visual treatment upgrade plus a few small UX additions.
