Shape-of-solution decisions baked into this mockup (override before code if any feel wrong)
  • 3 sub-shapes for v1: network-topology click, OSI stack click, cable/connector grid click. Each tested below.
  • Curated bank, not AI-generated: ~8–10 hand-authored questions for v1 (inline SVGs + answer keys + explanations). Same legal/quality discipline as the curated exemplar bank — never ingest paid content. Future: AI generation if it works for MCQs.
  • Single-click + Submit: matches our other PBQ types (multi-select, order, topology). User clicks a region → tentative selection ring → Submit grades it. Allows changing pick before commit.
  • Where they appear: regular quiz mix + exam simulator (via existing injectPBQs). NOT in diagnostic — diagnostic stays MCQ-only to honor the 30-min time budget (per v4.81.0 design).
  • Revisit-aware: integrates with the v4.82.0 revisit infrastructure — clicking a dot re-renders with the previous pick highlighted + Submit re-enabled, same as multi-select / order.
  • Schema: { type: 'hot-area', subShape: 'topology'|'osi'|'cable-grid', question, svgInline, regions: [{id, label, isCorrect}], answer: 'r3', explanation }. Stored in HOT_AREA_BANK array, injected into questions same way injectPBQs handles cli-sim and topology.
Example 1 · Network topology click · pre-pick
Q4 Hot Area · Topology Exam Level
A user reports they cannot reach the internet from the LAN. ICMP from PC1 to the default gateway succeeds, but ICMP from PC1 to 8.8.8.8 fails. Click the device most likely misconfigured.
Click a device to select your answer
SVG region pattern Each clickable region is a <g class="hot-region" data-region="X"> wrapping the visible shape + label. CSS hover rules live on .hot-region:hover children. The same group becomes .is-picked / .is-correct / .is-wrong / .is-dimmed / .is-reveal-correct via JS class toggling — no DOM rebuilding.
Example 1 · revealed · user picked R1, correct was FW1
Q4 Hot Area · Topology Exam Level
A user reports they cannot reach the internet from the LAN. ICMP from PC1 to the default gateway succeeds, but ICMP from PC1 to 8.8.8.8 fails. Click the device most likely misconfigured.
✗ Wrong — correct device: FW1 (firewall)
PC1 reaching its default gateway means LAN, switch, and router are all functional at L2/L3. Failure occurs only when traffic crosses the firewall — most likely cause is an outbound rule denying ICMP or a missing NAT/PAT translation. R1 has Layer 3 routing working (it forwarded to FW1), so it is not the misconfigured device.
Pedagogical pattern matches our other PBQs Picked region = red ring (your wrong choice). Correct region = solid green. Other 3 dimmed (not relevant to this question). Same vocabulary as .option.wrong / .option.correct / .option.dimmed from MCQ + multi-select. Reduces user mental load — the visual language of "right vs wrong" is already learned.

Example 3 · cable connector grid · revealed (correct first try)
Q12 Hot Area · Cable Foundational
Which connector would you use to terminate single-mode fiber for long-haul SFP+ uplinks in a data centre?
🔌 RJ-45
🔗 RJ-11
💎 LC
🟦 SC
📐 ST
🔘 F-Type
📺 BNC
⚡ USB-C
✓ Correct! · LC connector
LC (Lucent Connector) is the small-form-factor fibre connector used in SFP and SFP+ transceivers. It supports both single-mode and multi-mode fibre. SC connectors are bulkier and typically used in older termination panels; ST is bayonet-style and rarely seen in modern data-centre uplinks.
Cable grid uses emoji as v1 placeholder iconography For ship-today scope: emoji are good-enough visual differentiation without us having to author or license SVG icons for 8+ connector types. Curated bank covers ~8 connectors. If/when we want true visual realism, swap the emoji span for an inline SVG per connector — same DOM shape, no logic changes.
Revisit · OSI stack · user picked L4 originally, now revisiting
Q7 · revisiting Hot Area · OSI Foundational
At which OSI layer does ARP (Address Resolution Protocol) operate?
L7 · Application
L6 · Presentation
L5 · Session
L4 · Transport
L3 · Network
L2 · Data Link
L1 · Physical
✗ Wrong — ARP operates at Layers 2 / 3 (boundary protocol)
ARP is the canonical OSI-boundary protocol — it lives at Layer 2 (uses MAC addresses, broadcast frames) but resolves Layer 3 IP addresses. CompTIA accepts both L2 and L3 as correct answers; some sources call it "Layer 2.5". The most common single-layer answer on the exam is Layer 2 (Data Link).
✏️ Revisiting — pick again to change your answer. Score updates; streak stays.
Revisit + ambiguous-answer handling Two-fold: (1) Revisit works the same as v4.82.0 — clicking a dot re-renders with previous pick + the v4.82.0 yellow banner + re-enabled Submit. The re-pick branch in submitHotArea(q) updates the log entry in place, recomputes counters, leaves streak alone. (2) ARP is genuinely ambiguous between L2 / L3, so this question's schema lists BOTH as correct (regions: [..., {id:'L3', isCorrect:true}, {id:'L2', isCorrect:true}, ...]). Same dual-correct pattern works for any question where CompTIA accepts multiple answers.
Implementation summary · same-day ship scope
What ships today (~6 surfaces, modest LOC)
  1. HOT_AREA_BANK — curated array of ~8 questions: 3 topology, 3 OSI, 2 cable. Hand-authored SVGs + answer keys + explanations. Topic-tagged so it injects into the right quizzes.
  2. renderHotArea(q, box, ans) + submitHotArea(q) + _restoreAnsweredHotAreaState(q, entry) — three new helpers. render() dispatch already exists per the existing pattern.
  3. injectPBQs extension — current code injects 1 cli-sim/topology PBQ per quiz of n>=10, 2 in exam. Extend to also pull from HOT_AREA_BANK in the same rotation.
  4. getQType — already returns q.type, no change needed.
  5. CSS: .hot-region hover + state classes (svg subShape), .osi-layer + .cable-card for non-svg subShapes, full reduced-motion gate.
  6. Tests: UAT vm-fixtures for submitHotArea (pick → Submit → reveal markers + log entry shape), _restoreAnsweredHotAreaState (revisit). Playwright E2E: seed quiz with hot-area Q via _testInjectQuiz, click region, click Submit, assert reveal markers.
