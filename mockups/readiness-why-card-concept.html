Design notes
  • Sits directly under the existing Exam Readiness card on the Analytics page (same width, same theme).
  • 4 component bars use a tier color palette: good (≥80) · ok (50-79) · low (25-49) · critical (<25).
  • The diagnosis paragraph is dynamically generated. Algorithm: pick the lowest-contribution component, name how many topics/factors are pulling it down, suggest the cheapest fix.
  • The 2 action cards are derived from the existing whatIf attribution data already computed in getReadinessScore() — no new math needed. The "primary" (orange) card is the highest-leverage option.
  • "+14 pts" / "+8 pts" labels: computed from the same formula. Click-throughs go to existing focused-quiz flows (refresh stale uses the v4.81.15 stale-topic rotation, weakest uses the existing weak-spots drill).
  • Mobile (<540px): card stacks vertically, action cards stack vertically, bar labels shrink. No new components needed.
  • Implementation cost estimate: ~150 LOC in app.js for _renderAnaWhyScore(), ~120 LOC in styles.css, no new state — reads getReadinessScore() output directly.
