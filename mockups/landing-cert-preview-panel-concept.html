Implementation notes scope · files · risks

What ships

  • New <section class="proof-of-product"> in landing/index.html, inserted between #certs and #pass-guarantee.
  • Static SVG art for TB 2D, TB 3D, Packet Trace, Acronym Blitz (mockup uses inline SVG; ship will keep them inline so there's no extra image asset to host/cache).
  • Tabs are pure CSS + a tiny script for swap — no framework, ~30 LOC of vanilla JS to handle tab clicks + per-cert content swap. Auto-rotate after 6s of idle, manual click pauses.
  • Mosaic uses CSS grid — collapses to single column at ≤880px (matches existing landing breakpoints).

Per-cert content matrix

  • Network+ (LIVE) → TB 2D · TB 3D · Packet Trace · CTA "Try the topology lab →" linking to networkplus.certanvil.com/?cert=netplus#tb
  • Security+ (ACTIVE) → Acronym Blitz live tile (wide) · Attack-to-Mitigation preview (#301) · IR Phase Sorter preview (#303). CTA "Try Acronym Blitz →" linking to networkplus.certanvil.com/?cert=secplus#drills
  • AZ-900 / CCNA / AWS / AZ-104 (COMING SOON) → single full-width "being forged" placeholder tile + "Notify me ↗" CTA. Honest unbuilt state.

Why this is the right placement

  • Visitors see the cert tiles first (decision-making layer), then the proof-of-product panel (validation layer). Sequence matches buyer intent: "which cert?" → "what do I actually get?"
  • Sits before the Pass Guarantee section, which is trust-building. Order reads: pick → see → trust → buy.
  • Tab pattern means the section scales — adding AZ-900 later is a single tab + content block, no new section.

Risks & mitigations

  • SVG rendering across browsers — keeping art simple (rects + lines + text), no fancy filters. Fallback: img tag with raster.
  • Auto-rotate annoyance — 6s feels long enough; user click pauses. Single regression-guard test in landing's existing JS.
  • Tab content jump — fixed min-height on .pp-mosaic so swapping tabs doesn't shift surrounding content.
