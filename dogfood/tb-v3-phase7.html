Founder dogfood — TB v3 Phase 7 (3D mode)

Branch: feat/tb-v3-phase7 · Target version: v6.4.0 · Open in: Topology Builder v3

10-step manual smoke test. Run through each step in order; jot any glitch, surprise, or idea in the notes box. If a step fails: do NOT mark complete — flag for fix before ship.

Step 1: Activate 3D mode from Design state

Open Topology Builder v3. With no topology on canvas, click the 3D modebar pill.

Expected: scene tilts back ~50°, perspective vanishing point visible. Modebar pill highlights.

Step 2: 3D mode with existing topology

Drag a few devices onto the canvas, connect with cables, then click 3D pill again.

Expected: all devices stand vertically from the table plane; cables lie flat. No overlap glitches.

Step 3: Start trace in 3D

Pick a source device + destination, click Start in the floating control strip.

Expected: packet appears at source, glides along cable in 3D. Strip status updates with "Hop 1 of N — …".

Step 4: Source device cascade (encap)

Watch the source standing device as the packet arrives.

Expected: packet rises into the device, 7 OSI rows appear inside the device card, then cascade L7 → L1 (rows light top-down, ~560ms total). Packet then falls back to cable.

Step 5: Intermediate router cascade (decap-up + re-encap-down)

Step to the next hop (router or l3-switch). Watch the standing device.

Expected: packet rises, then cascade fires L1 → L2 → L3, brief pause, then L3 → L2 → L1. ~680ms total. Visible "the device touched layers up to L3" beat.

Step 6: Destination device cascade (decap)

Step to the final destination hop.

Expected: packet rises into destination device, cascade L1 → L7 (~560ms). Packet then falls.

Step 7: Failure path in 3D

Restart with a deliberately broken topology (disconnect a cable mid-route, or pick endpoints with no path). Click Start.

Expected: packet rises into the failing device, cascade fires up to the failing layer, that row pulses red 3x. Device card pulses with red rim glow. Packet stays trapped inside (no fall).

Step 8: Esc cleanup

While in 3D mode (mid-trace or idle), press Esc.

Expected: 3D mode closes cleanly. Scene flattens. Floating control strip disappears. No lingering 3d-open body class. Right-rail trace/OSI panel restored if it was open prior to 3D entry.

Step 9: Reduced-motion mode

macOS: System Settings → Accessibility → Display → Reduce motion. Reload the app. Enter 3D mode and run a trace.

Expected: scene does NOT tilt (stays flat). Packet does NOT rise/fall. Cascade rows appear instantly (no per-row stagger). Failure glow becomes static red border. All trace logic still works — just no motion drama.

Step 10: Mode round-trip (3D ↔ Trace ↔ OSI)

Start a trace in 3D, advance 2 hops. Click Trace pill — should see same trace in 2D with right-rail hop chips. Click OSI pill — should see Phase 6 panel with same hop selected. Click 3D pill — back to 3D scene, same trace state.

Expected: no orphan animations, no flicker, no state loss. Floating strip appears/disappears correctly on entry/exit of 3D.
